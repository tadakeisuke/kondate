# 献立管理リポジトリ (Kondate)

## 概要

AI献立エージェントのための **Single Source of Truth (SSoT)** として機能するデータリポジトリです。

このリポジトリでは以下の情報を一元管理します:
- 🥕 **冷蔵庫・パントリーの在庫**
- 👨‍👩‍👧‍👦 **家族の好みやアレルギー情報**
- 📅 **献立の履歴と評価**
- 📖 **お気に入りレシピ**
- 🏪 **近隣スーパーの情報と特売日**

AIエージェントがこれらの情報をコンテキストとして、以下を提案します:
- 今日の献立
- 買い物リストと最適な店舗
- 食材の使い切りアイデア
- 週間献立プラン

**注意**: コミット履歴には冷蔵庫の中身や食事内容が記録されます。プライベートリポジトリでの運用を推奨します。

## リポジトリ構成

```
kondate/
├── data/
│   ├── inventory/          # 在庫管理
│   │   ├── fridge.yaml    # 冷蔵庫の中身
│   │   └── pantry.yaml    # パントリー（常温保存）
│   ├── family/             # 家族情報
│   │   └── members.yaml   # 家族構成と好み
│   ├── history/            # 献立履歴
│   │   └── YYYY-MM.yaml   # 月次履歴
│   ├── recipes/            # レシピ集
│   │   └── favorites.yaml # お気に入りレシピ
│   └── shops/              # スーパー情報
│       └── supermarkets.yaml
├── docs/                   # ドキュメント
│   └── usage-guide.md     # 使い方ガイド
├── schemas/                # データ定義
│   └── data-schema.md     # スキーマドキュメント
└── README.md
```

## クイックスタート

### 1. 初期設定

各YAMLファイルをあなたの家庭の情報に書き換えてください:

```bash
# 冷蔵庫の中身を記録
vim data/inventory/fridge.yaml

# 家族の好みを設定
vim data/family/members.yaml

# 近くのスーパー情報を追加
vim data/shops/supermarkets.yaml
```

### 2. AIエージェントへの質問例

```
「今日の夕飯は何がいい？」
「明日の買い物リストを作って」
「賞味期限が近い食材で作れる料理は？」
「来週の献立を考えて」
```

### 3. データメンテナンス

**日次**:
- 夕食後に献立と評価を記録
- 買い物後に在庫を更新

**週次**:
- 週末に週次サマリーを追加
- 賞味期限をチェック

## 主な機能

### 献立提案
- 在庫状況から最適な献立を提案
- 家族の好みとアレルギーを考慮
- 賞味期限が近い食材を優先活用
- 献立履歴から重複を避ける

### 買い物最適化
- 必要な食材をリストアップ
- スーパーの特売日を考慮
- 最適な買い物ルートを提案
- 予算内での購入プラン

### 栄養バランス管理
- 週間の料理ジャンル分析
- タンパク質源のバリエーション確認
- 野菜摂取量の可視化

### 食品ロス削減
- 賞味期限アラート
- 使い切りレシピの提案
- 残り物活用アイデア

## ベストプラクティス

1. **正確な記録**: 在庫は正確に記録（AIの提案精度に直結）
2. **率直な評価**: 家族の反応を正直に記録
3. **定期更新**: 週1回はデータを見直す
4. **継続運用**: 長期間のデータ蓄積でAI精度が向上

## データ形式

すべてのデータはYAML形式で管理されています。人間が読み書きしやすく、AIも理解しやすい形式です。

例:
```yaml
vegetables:
  - name: にんじん
    quantity: 2
    unit: 本
    purchase_date: 2025-11-23
    expiry_date: 2025-12-07
```

## プライバシーとセキュリティ

このリポジトリには家族の個人的な情報（食の好み、健康情報、生活パターン）が含まれます:
- **プライベートリポジトリ**として運用してください
- 共有する場合は情報をマスクしてください
- 定期的にバックアップを取ることを推奨します

## 設計思想

このリポジトリは**データ駆動型の献立管理**を実現します。すべての情報を構造化されたデータとして保存することで:
- AIが一貫性のある提案を生成
- 時系列での分析が可能
- 家族の好みの変化を追跡
- 効率的な在庫管理と食品ロス削減

Git履歴により、献立の変遷や在庫の推移を振り返ることもできます。
